WordPress : URL sans domaine

6 septembre 2013
 Partager sur TwitterTweeter  Partager sur Facebook  

La configuration de WordPress vous oblige à saisir une URL complète pour l’adresse web de votre site. Cela empêche d’accéder au même site depuis plusieurs adresses, par exemple mondomaine.com, localhost, ou 192.168.1.42, ce qui peut être gênant si par exemple on héberge son site à la maison et que l’IP publique n’y est pas routée.

Pour contourner ce problème, si votre site a comme adresse mondomaine.com/foobar/, éditez la table wp_options de votre base de données (vous pouvez utilisez phpMyAdmin avec une base MySQL, ou sqlitebrowser avec une base SQLite) et remplacez les entrées siteurl et home par /foobar, au lieu de http://mondomaine.com/foobar .

Si votre site est à la racine du domaine, remplacez /foobar par / .
